# 알려진 에러 패턴 DB

실기기 테스트(2026-04-15~16)에서 확인된 에러와 대응 방법.

## 에러 분류 계층

| Level | 설명 | 예시 |
|-------|------|------|
| Level 1 | 사용자 자가 해결 (재시도, 설정 변경, 재부팅) | loadingInterrupted, extensionDisabled |
| Level 2 | 앱 초기화 필요 (삭제 → 재부팅 → 재설치) | SQLite CONSTRAINT, DB 잔존 |
| Level 3 | 코드 수정 필요 (개발자 대응) | entriesOutOfOrder, duplicateEntries |
| Level 4 | 아키텍처 변경 필요 (K값, N값 조정) | maximumEntriesExceeded, 메모리 부족 |
