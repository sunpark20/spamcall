## 성공 흐름 분석 (#1)
1. Settings에서 20개 켬 → App Group 버그로 빈 로드 (즉시 완료)
2. 앱 코드 수정 → K=175만 하드코딩 (App Group 제거)
3. 앱에서 reload → **증분 모드**로 175만 등록 (비증분 빈 로드 후 증분)
4. 18개 전부 성공, 374초, 전화 차단 확인

## 핵심 발견

### 시간 한도
- iOS 익스텐션 실행 시간 한도: **~30초**
- 비증분 175만: ~21초 (여유 9초) ✅
- 비증분 350만: ~27초에서 중단 ✗
- 증분 200만: ~28초에서 중단 ✗
- 증분 175만: ~21초 (성공 #1에서 확인) ✅
- 처리 속도: ~83,000건/초 (비증분), 증분은 약간 느림

### App Group
- app→extension: **불안정** (때때로 동작, 때때로 안 됨)
- extension→app: **동작 안 함**
- 하드코딩이 유일하게 신뢰 가능한 방법

### 증분 로딩
- 빈 비증분 후 증분 추가: **175만까지 동작** (성공 #1)
- 같은 익스텐션 반복 증분: **SQLite 에러로 실패** (F3)
- removeAllBlockingEntries 후 재등록: **시간 초과** (F4)

### Settings 동시 활성화
- 50개 동시: **전부 실패** (F5)
- 20개 동시: 빈 로드 버전에서 성공 (App Group 버그로 실제로 빈 로드)

## 현재 검증된 최적 전략
- Settings 토글 → 빈 로드 (비증분에서 아무것도 안 함)
- 앱에서 순차 reload → 증분으로 175만씩 등록
- K=175만, N=58개로 1억 달성 가능
